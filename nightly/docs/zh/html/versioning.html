了解版本信息¶

每个 EduMIPS64 构建都会显示一个版本标识符，让您始终清楚自己正在运行的版本。 本页说明该标识符的含义及查找方式。

版本字符串¶

在正式发布时，版本号是一个简单的数字，例如 1.4.1。

在两次发布之间生成的构建版本类似于 1.4.0-74-geec1768。 从左到右依次解读：

  • 1.4.0 — 此构建所基于的最近一次正式发布版本。

  • 74 — 该正式发布之后所做的更改数量。

  • geec1768 — 本次构建的简短唯一标识符。

您无需记住该标识符。其主要用途是在提交错误报告时，让维护者能够精确定位 您所使用的*确切构建*，从而无需猜测即可复现问题。

在哪里查找版本¶

  • 桌面（Swing）应用程序 — 显示在窗口标题栏以及 帮助 → 关于 对话框中。

  • 命令行 — 使用 --version 标志运行 JAR 文件。

  • 网页应用程序 — 显示在帮助对话框的 About 标签页中，以及顶部工具栏的 "Web Version" 标签旁边。

我正在使用哪个网页构建版本？¶

位于 https://web.edumips.org 的生产网页应用包含稳定版本。 只有当维护者明确推送新发布时，它才会更新，因此是日常使用的最佳选择。

此外还有其他网页构建版本，通过工具栏中 "Web Version" 标签旁显示的 小彩色徽章加以标识：

  • 无徽章 — 您正在使用正常的生产构建，即 https://web.edumips.org 上的稳定版本。

  • ``NIGHTLY``（橙色徽章）— 每晚从最新开发代码自动重新构建的版本。 它始终包含最新功能，但可能不稳定。将鼠标悬停在徽章上可查看确认此信息的提示。

  • ``PR #N``（黄色徽章）— 针对 GitHub 上某个特定拟议更改（pull request） 生成的临时预览版本。点击徽章可打开对应的 pull request 页面。

  • ``dev``（蓝色徽章）— 有人正在本地计算机上直接运行开发构建版本。

简而言之：无徽章 → 稳定生产版；NIGHTLY→ 最新功能但可能不稳定；PR #N / dev→ 临时或预览构建。

查看以前的版本¶

每个已发布的网页版本都作为不可变的快照保留，并继续可用。您可以从网页应用程序的帮助对话框中查看所有保留版本的列表，并打开更早的版本：

  1. 单击帮助→ 关于（网页应用工具栏中）。

  2. 在"About"标签页中查找Previous Versions部分。

  3. 您将看到带有日期的已发布版本列表。

  4. 当前版本标有标签，便于您了解哪个版本是活跃版本。

  5. 单击任何早期版本以在新标签页中打开它 — 您当前的工作将保留在原始标签页中。

如果您需要比较版本、验证早期版本中的错误，或使用更适合您需求的版本，这会很有用。
