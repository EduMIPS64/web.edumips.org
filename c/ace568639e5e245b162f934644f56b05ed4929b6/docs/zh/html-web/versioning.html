了解版本信息¶

每个 EduMIPS64 构建都会显示一个版本标识符，让您始终清楚自己正在运行的版本。 本页说明该标识符的含义及查找方式。

版本字符串¶

在正式发布时，版本号是一个简单的数字，例如 1.4.1。

在两次发布之间生成的构建版本类似于 1.4.0-74-geec1768。 从左到右依次解读：

  • 1.4.0 — 此构建所基于的最近一次正式发布版本。

  • 74 — 该正式发布之后所做的更改数量。

  • geec1768 — 本次构建的简短唯一标识符。

您无需记住该标识符。其主要用途是在提交错误报告时，让维护者能够精确定位 您所使用的*确切构建*，从而无需猜测即可复现问题。

在哪里查找版本¶

  • 桌面（Swing）应用程序 — 显示在窗口标题栏以及 帮助 → 关于 对话框中。

  • 命令行 — 使用 --version 标志运行 JAR 文件。

  • 网页应用程序 — 显示在帮助对话框的 About 标签页中，以及顶部工具栏的 "Web Version" 标签旁边。

我正在使用哪个网页构建版本？¶

位于 https://web.edumips.org 的生产网页应用包含稳定版本。 只有当维护者明确推广（promote）新构建时，它才会更新，因此是日常使用的最佳选择。

此外，开发代码的每个提交也会被构建并发布到其自己的永久地址 https://web.edumips.org/c/<commit>/ 下。工具栏中 "Web Version" 标签旁的 小彩色徽章会告诉您当前所处的构建类型：

  • 无徽章 — 您正在使用正常的生产构建，即 https://web.edumips.org 上的稳定版本。

  • ``ARCHIVED`` — 您正在查看由 /c/<commit>/ 提供的某个特定提交的构建。 它可能是先前推广过的版本，也可能是等待推广的候选版本；无论哪种情况，它都不是 正在运行的生产站点。将鼠标悬停在徽章上可查看确认此信息的提示。

  • ``PR #N``（黄色徽章）— 针对 GitHub 上某个特定拟议更改（pull request） 生成的临时预览版本。点击徽章可打开对应的 pull request 页面。

  • ``dev``（蓝色徽章）— 有人正在本地计算机上直接运行开发构建版本。

简而言之：无徽章 → 稳定生产版；ARCHIVED→ 某个特定提交的构建；PR #N / dev→ 临时或预览构建。

浏览与切换版本¶

所有保留的网页构建——既包括已推广的发布版本，也包括待推广的候选版本——都以其 提交标识符列在同一处。请从帮助对话框打开该列表：

  1. 单击帮助→ 关于（网页应用工具栏中）。

  2. "About" 标签页会显示两个列表：

    • Promoted versions — 手动推广的发布版本，以醒目方式显示。 当前活跃的版本标记为 current。

    • Candidate builds — 尚未推广的较新构建。所有待推广的候选版本都会显示 （通常只有少数几个）。

  3. 单击任意条目即可在新标签页中打开该精确构建——您当前的工作会保留在原始标签页中。

保留哪些版本，以及保留多久¶

  • 已推广的版本会作为不可变快照永久保留，因此您随时可以回到任何发布版本。

  • 候选构建会一直保留，直到发生一次推广。当某个构建被推广时，排在它之前且 从未被推广的候选版本会被移除；比已推广构建更新的候选版本会被保留，并持续可用， 直到它们自身被推广或被取代。

这样可使列表保持精简：每个已推广的发布版本，加上少数仍在等待推广的构建。
